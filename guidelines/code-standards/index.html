Code Standards

Under development

Naming

Functions, mixins, and variables should be declared with all lowercase letters and hyphens instead of underscores. Exceptions made for outside vendor libraries, such as bower components.

No IDs

ID selectors should never be used for the purposes of styling an element, as it leads to overly specific styles that aren't easily shared with other elements. IDs should only be used when absolutely needed by JavaScript, form elements, or WAI-ARIA roles.

Classes

The Styleguide follows the BEM syntax.

Whitespace

Make sure you are using .editorconfig

  • Use one tab for indentation. No spaces.
  • Insert a space after comma separated items in a list.

Comments

Comments should be used to provide more information.

Shorthand

Prefer the shortest shorthand form possible for properties that support it (i.e margin, padding, border)

Omit unit lengths on zero values.

Bad

margin: 0px;

Good

margin: 0;
